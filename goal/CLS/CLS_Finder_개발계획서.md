3. §5 의 파이프라인을 단계별 함수/도구로 구현한다. 각 단계는 독립적으로 테스트 가능해야 한다.

## 1. 프로그램 개요 및 목표

### 1.1 한 줄 정의
타이트 바인딩 해밀토니안과 격자 구조를 입력받아, **플랫 밴드를 자동 검출하고 → 해당 컴팩트 국소 상태(CLS)를 해석/수치적으로 도출하며 → 특이/비특이(singular/non-singular) 여부를 판정하고 → 실공간 격자 위에 CLS를 시각화**하는 Python 프로그램.

### 1.2 전체 입출력

| 항목 | 내용 |
|---|---|
| **입력** | (a) 타이트 바인딩 해밀토니안 — hopping 리스트 또는 기호적 `H(k)` 행렬, (b) 격자 구조 — 차원 `d`, 기본 병진 벡터 `a_l`, 단위 셀 내 오비탈/사이트 위치와 라벨 |
| **출력** | (1) 밴드 구조 플롯, (2) 플랫 밴드 목록(에너지 ε₀, 축퇴도), (3) 각 플랫 밴드의 Bloch 고유벡터, (4) singular/non-singular 분류와 특이점 `k₀`, (5) CLS 실공간 진폭 `A_{0,R}` (해석식 + 수치값), (6) singular인 경우 NLS/NPS 정보, (7) 격자 위에 CLS가 표시된 그림, (8) 요약 리포트(JSON + 사람이 읽는 텍스트) |

### 1.3 기능 범위
- 차원 `d = 1, 2, 3` 지원.
- 단위 셀당 오비탈 수 `Q` 임의 (2×2 특수 케이스 + 일반 `Q×Q` 둘 다 구현).
- 유한 범위(finite-range) hopping만 가정 (논문 전제). 입력에서 이를 검증.
- 단일 플랫 밴드 및 **축퇴된 다중 플랫 밴드**(kagome-3 사례) 처리.

## 2. 이론적 배경의 알고리즘적 요약

에이전트가 구현해야 할 수학적 사실을 "코드로 옮길 수 있는 형태"로 정리한다.

### 2.1 Bloch 고유상태와 CLS
밴드 `n`의 Bloch 상태는 식 (2.1):
```
|ψ_{n,k}⟩ = (1/√N) Σ_R Σ_{q=1..Q} e^{ik·R} v_{n,k,q} |R,q⟩
```
플랫 밴드 내 상태들의 선형결합으로 CLS `|χ_R⟩`를 구성하며, 실공간 진폭은 식 (2.3):
```
A_{R,R'} = (c_χ/√N) Σ_{k∈BZ} α_k e^{ik·(R'-R)} v_k
```
이는 `α_k v_k` 의 **역 푸리에 변환**이다.

### 2.2 FSBP 조건 (가장 중요한 자료구조 근거)
`|χ_R⟩` 가 컴팩트하려면 `α_k v_{k,q}` 의 각 성분이 **유한 블로흐 위상합(Finite Sum of Bloch Phases, FSBP)** 이어야 한다 — 식 (2.4):
```
α_k v_{k,q} = Σ_{m_1..m_d} f^{(q)}_{m_1..m_d} · exp( i Σ_l m_l k_l·a_l )
```
즉 변수 `X_l = exp(i k_l·a_l)` 에 대한 **다변수 복소 Laurent 다항식**이다. 계수 `f` 가 곧 CLS의 실공간 진폭이다. → **모든 핵심 연산은 Laurent 다항식 대수로 환원된다.**

### 2.3 커널 방정식
`x_k = α_k v_k` 는 비규격화 고유벡터이고 식 (2.5):
```
H̄_k x_k = (H_k − ε₀ I) x_k = 0
```
의 해다. `H_k` 의 모든 성분이 FSBP이므로 (유한 hopping), 이 동차 방정식은 **FSBP 형태의 해를 반드시 갖는다** (App. A, 수학적 귀납법 증명). → 수치 커널 해법의 정당성.

### 2.4 완비성 판별식
`N` 개의 병진 CLS 사본의 선형독립 여부는 식 (2.6):
```
D ∝ Π_{l=1..N} α_{k_l}
```
- `α_k` 가 **모든 k에서 0이 아니면** → `N`개 CLS가 완비집합 → **비특이(non-singular) 플랫 밴드**.
- 어떤 `α_k` 선택을 해도 **어떤 k₀에서 반드시 0** 이면 → 완비성 붕괴 → **특이(singular) 플랫 밴드**.

### 2.5 특이성 ⟺ 파동함수 부동 불연속성
공통 인수가 없는 FSBP 벡터 `w_k = (w_{k,1},…,w_{k,Q})` 에 대해:
- `v_k` 가 `k₀`에서 불연속 ⟺ **모든** `w_{k,q}` 가 `k₀`에서 동시에 0 ⟺ `α_k(k₀)=0`.
- **따라서 알고리즘적 판정법:** "공통 인수를 제거한 FSBP 고유벡터 `w_k` 의 모든 성분이 동시에 0이 되는 `k₀`가 BZ 안에 존재하는가?" → 존재하면 singular.
- **1차원 플랫 밴드는 항상 비특이**다 (논문 정리). → `d=1` 분기에서 분류를 단축할 수 있다.

### 2.6 일반 α_k 도출 공식 (해석 경로의 핵심) — 식 (5.1)
가이드 오비탈 인덱스 `p` 를 골라:
```
α_k = det( H̄_k^{(p,p)} ) / v_{k,p}
```
`H̄_k^{(p,p)}` 는 `H̄_k` 에서 `p`행·`p`열을 제거한 `(Q−1)×(Q−1)` 소행렬. 그 결과 (식 5.2~5.3):
```
x_{k,p}  = det( H̄_k^{(p,p)} )
x_{k,j≠p} = [ −Adj(H̄_k^{(p,p)}) · (H̄_k|_{1,p}, …, H̄_k|_{Q,p})^T ]_j
```
**분모 소거의 정당성:** Adjugate를 Faddeev–LeVerrier / Cayley-Hamilton (식 5.4)로 전개하면 `det` 인수가 `α_k` 의 분모와 약분되어 순수 FSBP가 남는다. → §6.3 에서 이 알고리즘을 구현.

### 2.7 역설계(Inverse engineering) — 식 (6.18~6.26)
원하는 CLS(=`α_k v^{(0)}`)를 먼저 정하고, Gram-Schmidt로 직교 고유벡터 `v^{(q)}` 들을 만든 뒤:
```
H_k|_{ij} = Σ_{q=1..Q−1} E^{(q)}_k · v^{(q)}_{k,i} · v^{(q)*}_{k,j}     (6.21)
E^{(q)}_k = F_k · (α^{(q)}_k)^2                                       (6.26)
```
로 해밀토니안을 합성한다. → **선택 기능**(§5.9)으로 구현 권장.

### 3.2 AI 에이전트 실행 모델
프로그램은 **결정론적 계산 도구들 + 이를 지휘하는 LLM 에이전트** 구조로 만든다. 무거운 수치·기호 계산은 절대 LLM에게 시키지 않고 순수 Python 도구가 수행하며, LLM은 **판단·분기·예외처리**만 담당한다.

에이전트에 노출할 도구(tool) 목록:
- `parse_input(spec)` → `Lattice`, `H_of_k`
- `compute_bands(H_of_k, grid)` → 밴드 에너지 배열
- `detect_flat_bands(bands, tol)` → 플랫 밴드 인덱스·에너지·축퇴도
- `extract_eigenstate(H_of_k, eps0)` → `v_k`(기호) 또는 수치 고유벡터장
- `classify_singularity(w_k, lattice)` → `{singular: bool, k0: [...]}`
- `extract_cls_analytic(H_of_k, eps0, p)` → `x_k`(FSBP), `A_{0,R}`
- `extract_cls_numeric(H_of_k, eps0, grid)` → `A_{0,R}`(수치)
- `minimize_cls(A)` → 최소 면적 CLS
- `build_noncontractible(...)` → NLS/NPS
- `plot_lattice_cls(lattice, A)` , `plot_bands(bands)`
- `make_report(...)`

에이전트가 내려야 하는 **판단 지점**(이 때문에 LLM이 필요하다):
1. 가이드 인덱스 `p` 선택 — `v_{k,p}` 가 전 k에서 0이 되지 않는 성분을 골라야 함. 실패 시 다른 `p` 재시도.
2. 해석 경로 vs 수치 경로 결정 — 기호 계산이 시간 초과/실패하면 수치로 폴백.
3. 축퇴 플랫 밴드 발견 시 — 모멘텀 의존 혼합(kagome-3 식 G14~G15)을 시도할지 결정.
4. 검증 — 해석 결과와 수치 결과 불일치 시 재계산·경고.

> 에이전트 시스템 프롬프트에는 §2 의 수식 요약과 §10 의 주의사항을 반드시 포함시킨다.

## 4. 입력 사양 (Input Specification)

입력은 **단일 JSON(또는 Python dict)** 으로 받는다. 두 가지 해밀토니안 입력 방식을 모두 지원한다.

### 4.1 격자 구조 스키마
```json
{
  "dimension": 2,
  "primitive_vectors": [[1.0, 0.0], [0.0, 1.0]],
  "orbitals": [
    {"label": "A", "position": [0.0, 0.0]},
    {"label": "B", "position": [0.5, 0.5]}
  ]
}
```
- `position` 은 단위 셀 내 분수/직교 좌표 (시각화·실공간 좌표 계산용).
- 오비탈 순서가 곧 해밀토니안 행렬의 인덱스 순서 `q = 1..Q`.

### 4.2 해밀토니안 입력 — 방식 A: hopping 리스트 (권장)
```json
{
  "hoppings": [
    {"i": 0, "j": 1, "R": [0, 0],  "t": -1.0},
    {"i": 0, "j": 1, "R": [-1, 0], "t": -1.0},
    {"i": 0, "j": 0, "R": [0, 0],  "t":  2.0}
  ]
}
```
- `i, j` 는 오비탈 인덱스, `R` 은 정수 단위 셀 변위, `t` 는 복소수 가능(`{"re":…, "im":…}`).
- `i==j, R==0` 항은 onsite 포텐셜.
- 프로그램이 식 `H_k|_{ij} = Σ t · exp(i k·R)` 로 `H(k)` 를 자동 구성하고, **에르미트성**(`H_k = H_k†`)을 검증한다.

### 4.3 해밀토니안 입력 — 방식 B: 기호 행렬
```json
{
  "H_symbolic": [
    ["2 - 2*cos(ky)", "-(1 - exp(-I*ky))*(1 - exp(-I*kx))"],
    ["-(1 - exp(I*ky))*(1 - exp(I*kx))", "2 - 2*cos(kx)"]
  ]
}
```
- `sympy` 가 파싱. 변수명은 `kx, ky, kz` 로 고정.
- 파서는 즉시 `X_l = exp(I*k_l·a_l)` 치환을 적용해 §6.1 의 FSBP 표현으로 변환을 시도한다.

### 4.4 선택 옵션
```json
{
  "options": {
    "k_grid": [60, 60],         "flat_tol": 1e-6,
    "path": "auto",             "prefer": "analytic",
    "system_size_N": [12, 12],  "plot_range": [3, 3]
  }
}
```
- `path`: 밴드 구조용 고대칭점 경로. `"auto"` 면 격자 대칭에서 추정.
- `system_size_N`: 완비성의 짝/홀 효과(zigzag·checkerboard-II) 점검용.

## 5. 처리 파이프라인 (단계별 상세)

### 5.1 입력 파싱 및 H(k) 구성
1. 스키마 검증 — 차원, 오비탈 수 `Q`, hopping 인덱스 범위.
2. 방식 A 면 hopping → `H(k)` 합성, 방식 B 면 기호 행렬 직접 사용.
3. **에르미트성 검사**: `H_k − H_k†` 가 항등적으로 0인지 (Laurent 계수 비교). 실패 시 에러.
4. **유한 범위 검사**: 모든 hopping `R` 이 유계인지 확인.
5. 산출: `Lattice` 객체 + `H(k)`(MatrixPoly, §6.2).

### 5.2 밴드 구조 계산 및 플랫 밴드 검출
1. BZ를 `k_grid` 로 균일 샘플링.
2. 각 k에서 `H(k)` 를 수치 대각화 → 정렬된 에너지 `E_n(k)`.
3. 각 밴드의 BZ 전역 분산 `Var_n = ⟨E_n²⟩ − ⟨E_n⟩²` 계산.
4. `Var_n < flat_tol` 인 밴드를 **플랫 밴드 후보**로 표시, 에너지 `ε₀ = ⟨E_n⟩`.
5. **해석적 교차검증(가능 시):** `det(H_k − ε₀ I)` 가 Laurent 다항식으로서 **항등적 0**인지 검사 — 항등적 0이면 완전 평탄 확정.
6. **축퇴도 산출:** 같은 `ε₀` 를 갖는 플랫 밴드 개수를 기록 (kagome-3 = 2, Lieb = 3중 접점).
7. 고대칭점 경로를 따라 밴드 구조를 그릴 데이터 생성 (§5.8 에서 플롯).
8. 플랫 밴드가 없으면 그 사실을 리포트하고 종료.

### 5.3 플랫 밴드 고유상태 추출
- **해석 경로:** `(H_k − ε₀I) v = 0` 의 영공간을 `sympy.nullspace` 로 풀어 `v_k`(기호) 획득. 성분의 **공통 Laurent 인수를 제거**해 `w_k` 를 만든다 (§2.5 판정의 전제).
- **수치 경로:** 각 k 격자점에서 `ε₀` 에 해당하는 고유벡터를 수치로 추출. 단, 게이지(위상)가 k마다 임의이므로 **연속 게이지 고정** 절차 필요 — 이웃 k와의 내적이 양의 실수가 되도록 위상을 맞춘다.
- 축퇴 시: 영공간 차원 = 축퇴도. 기저 `{v^{(1)},…}` 를 모두 보관.

### 5.4 Singular / Non-singular 분류
입력: 공통 인수 제거된 FSBP 고유벡터 `w_k`.

1. **차원 단축:** `d == 1` 이면 즉시 `non-singular` 로 판정하고 종료 (논문 정리).
2. **밴드 접점 탐색:** 플랫 밴드가 분산 밴드와 만나는 `k` 를 §5.2 격자 데이터에서 찾는다 (에너지 갭이 `tol` 이하인 곳).
3. **공통영점 판정:** "모든 `w_{k,q}` 가 동시에 0" 인 `k₀` 를 구한다.
   - 해석: 다변수 다항식 연립 `{w_{k,q}=0}` 을 resultant 또는 Groebner basis(`sympy`)로 풀거나, `gcd` 가 비자명한지 점검.
   - 수치: 후보 `k`(고대칭점 + 접점 + 격자)에서 `Σ_q |w_{k,q}|²` 가 0에 수렴하는지 스캔.
4. 공통영점 `k₀` 가 **하나라도 있으면 → singular**, 그 `k₀` 목록을 기록. 없으면 → **non-singular**.
5. **축퇴 플랫 밴드 보정 (kagome-3 케이스):** 개별 `v^{(1)}, v^{(2)}` 가 서로 다른 `k` 에서 특이점을 가지면, **모멘텀 의존 선형결합**으로 특이점 제거를 시도한다 (식 G14~G15):
   ```
   w^{(1)} ∝ v^{(1)}/c1 − v^{(2)}/c2 ,   w^{(2)} ∝ v^{(1)}/c1 + v^{(2)}/c2
   ```
   혼합 후 두 기저 모두 공통영점이 사라지면 **전체를 non-singular 로 재분류**한다. 이는 "기하학적 좌절 ≠ 특이성" 을 보여주는 핵심 기능이므로 반드시 구현.
6. **완비성·짝홀 효과:** `system_size_N` 이 주어지면, 허용 모멘텀 `k = 2πn/N` 집합에서 `α_k` 가 0이 되는지 확인 — `k₀ = (π,…)` 가 짝수 N에서만 허용되는 효과(zigzag, checkerboard-II)를 리포트.
7. 산출: `{singular, k0_list, degeneracy, completeness_note}`.

### 5.5 CLS 도출 — 해석 경로 (`analytic.py`)
식 (5.1)~(5.3) 직접 구현.
1. `H̄_k = H_k − ε₀ I` (MatrixPoly).
2. `v_{k,p}` 가 항등적 0이 아닌 가이드 인덱스 `p` 선택. (에이전트가 결정; 실패 시 다른 p.)
3. `H̄_k^{(p,p)}` = `p`행·열 제거한 소행렬 (§6.2).
4. `x_{k,p} = det(H̄_k^{(p,p)})` — Laurent 다항식 행렬식.
5. `Adj(H̄_k^{(p,p)})` 를 **Faddeev–LeVerrier 알고리즘**으로 계산 (§6.3) → 분모 없는 순수 FSBP 보장.
6. 나머지 성분: `x' = −Adj(H̄_k^{(p,p)}) · (H̄_k|_{j,p})_{j≠p}`.
7. `x_k` 의 각 성분(Laurent 다항식)의 **계수 딕셔너리 `f^{(q)}_{m}` 가 곧 실공간 진폭** → `A_{0,R}` 직접 획득 (역 푸리에 변환 불필요, §2.2).
8. **검증:** `H̄_k · x_k ≡ 0` (Laurent 다항식 항등) 확인.
9. `Q == 2` 특수 케이스는 식 (A1) `x_k = (H̄_k|_{1,2}, −H̄_k|_{1,1})` 로 단축 가능.

### 5.6 CLS 도출 — 수치 경로 (`numeric.py`)
해석 경로가 막힐 때 또는 교차검증용.
1. 각 k 격자점에서 `H̄_k` 의 영공간 벡터 `v_k` 수치 계산.
2. `α_k` 후보를 잡아 (`α_k = (Σ|w_q|²)^{1/2}` 또는 §5.5의 `det` 식 수치값) `x_k = α_k v_k` 구성.
3. `x_k` 를 BZ 전역에서 **이산 푸리에 역변환** → `A_{0,R}`.
4. 임계값 이하 성분을 0으로 절단해 컴팩트 지지(support)를 추출.
5. 지지 영역이 유계임을 확인 (유계가 아니면 잘못된 `α_k` — 경고).
6. App. A 의 귀납적 커널 구성(식 A2~A3)을 수치로 구현해, 큰 `Q` 에서 안정적 폴백을 제공.

### 5.7 CLS 크기 최소화 (`reduce.py`)
식 (5.1)로 얻은 CLS는 종종 최소 단위보다 크다 (식 5.6 vs F3).
1. CLS를 Laurent 다항식 벡터로 보고, 병진 사본들의 **선형결합**으로 더 작은 지지를 탐색 — 식 (5.7) `A_{0,R} + A_{(1,0),R}` 형태.
2. 알고리즘: 성분 다항식들의 **공통 인수 추출 / 격자 환원**(다변수 다항식 GCD, 또는 작은 정수계수 결합 탐색)으로 최소 면적 지지를 찾는다.
3. 분리 가능한(decoupled) 사본이 나오면 그 중 하나를 최소 CLS로 선택.
4. 산출: 원본 CLS와 최소화 CLS를 둘 다 보관.

### 5.8 격자 및 CLS 시각화 (`viz/plot.py`)
1. **격자 그리기:** 기본 병진 벡터로 단위 셀을 `plot_range` 만큼 타일링, 각 오비탈을 위치에 점으로 표시(라벨별 색). hopping 리스트로 사이트 간 결합선을 그림(부호/세기에 따라 실선/점선·굵기).
2. **CLS 오버레이:** `A_{0,R,q}` 가 0이 아닌 사이트에 마커를 얹는다 — 색 = 진폭 부호, 크기 = `|진폭|`, 텍스트 = 진폭 수치. 컴팩트 지지 영역을 음영으로 감싼다 (논문 Fig. 1, 9 스타일).
3. **밴드 구조 플롯:** §5.2 의 고대칭점 경로 데이터로 `E vs k` 그림, 플랫 밴드를 강조색으로.
4. singular면 NLS/NPS도 함께 표시: `noncontractible.py` 가 특이축 모멘텀을 고정하고 1D 역변환으로 NLS를 구성(논문 §IV.A) — 한 축 국소·다른 축 격자 전체로 연장되는 형태.
5. 출력은 PNG + (가능하면) 인터랙티브 HTML.

### 5.9 (선택) 역설계 모드
사용자가 원하는 `α_k v^{(0)}` 를 입력하면 §2.7 (식 6.18~6.26)로 해밀토니안을 합성: Gram-Schmidt로 직교 기저 생성 → `E^{(q)}_k = F_k (α^{(q)}_k)²` → `H_k|_{ij} = Σ E^{(q)} v^{(q)}_i v^{(q)*}_j`. `α_k v^{(0)}` 의 영점 유무로 singular/non-singular를 사용자가 직접 설계.

## 6. 핵심 자료구조 및 알고리즘

### 6.1 `LaurentPoly` — FSBP 표현
- 내부 표현: `dict{ (m_1,…,m_d) : complex }` — 키는 정수 지수 튜플, 값은 계수.
- 구현 연산: 덧셈, 뺄셈, 곱셈, 스칼라곱, 상등(항등 0) 판정, 단일 항 인수 추출, 다변수 GCD, 특정 `k` 에서의 수치 평가, 켤레(`m → −m`, 계수 켤레).
- **이 dict 가 곧 CLS 실공간 진폭**이라는 점이 설계의 핵심 — §2.2.

### 6.2 `MatrixPoly` — 성분이 LaurentPoly인 행렬
- `det` (Laurent 다항식 행렬식, Leibniz 또는 분수 없는 Bareiss 변형 — 나눗셈 회피).
- `minor(p,p)` / `submatrix`, `trace`, 행렬곱, 켤레전치 `dagger`.
- 항등 0 행렬 판정.

### 6.3 Adjugate — Faddeev–LeVerrier
- `Adj(A) = det(A)·A^{-1}` 를 **나눗셈 없이** 트레이스 재귀로 계산 (식 5.4, Cayley-Hamilton + Newton 항등식).
- 반복: `M_0 = I; for r in 1..n: c_r = −tr(A·M_{r-1})/r; M_r = A·M_{r-1} + c_r I`. → `Adj(A) = ±M_{n-1}`.
- 모든 단계가 Laurent 다항식 곱·합·정수나눗셈뿐 → **FSBP 보존**. 이것이 §5.5 6단계의 정당성.

### 6.4 완비성 판정식 / NLS 구성
- `D ∝ Π α_{k_l}` (식 2.6) 를 허용 모멘텀 집합에서 평가.
- NLS: 특이 모멘텀의 한 축 성분을 고정한 유효 1D 해밀토니안에서 1D CLS를 구한 뒤 다른 축으로 연장 (§5.8-4).

## 10. 엣지 케이스 및 주의사항 (에이전트가 반드시 기억할 것)

- **게이지 임의성:** 수치 고유벡터는 k마다 위상이 임의다. 분류·푸리에 변환 전에 반드시 연속 게이지를 고정하라. 고정 실패 시 그것이 곧 부동 불연속성의 신호일 수 있다.
- **가이드 인덱스 `p`:** `v_{k,p}` 가 어떤 k에서 0이 되면 식 (5.1)이 정의되지 않는다. 0이 안 되는 `p` 를 찾을 때까지 바꿔라. 모든 `p` 가 실패하면 수치 경로로 폴백.
- **공통 인수 제거:** `w_k` 의 공통 Laurent 인수를 제거하지 않으면 §2.5 의 특이성 판정이 잘못된다. 분류 전 필수 전처리.
- **짝/홀 시스템 크기:** `k₀=(π,…)` 형 특이점은 N이 짝수일 때만 허용 모멘텀에 포함된다 (zigzag, checkerboard-II). 완비성 결론에 `system_size_N` 을 반영하라.
- **축퇴 플랫 밴드:** 개별 고유벡터가 특이해 보여도, 모멘텀 의존 혼합으로 비특이가 될 수 있다 (kagome-3). 축퇴를 발견하면 반드시 혼합을 시도한 뒤 분류하라.
- **연속 모델 vs 격자 모델:** 본 프로그램은 **유한 hopping 격자 모델**만 다룬다. 연속(continuum) 해밀토니안은 범위 밖 — 입력 검증에서 거부하거나 명시적으로 경고.
- **수치 vs 해석 불일치:** 정규화·전체 위상·CLS 크기 차이는 정상이다. 그러나 지지(support)의 위상(topology)이 다르면 버그 — 재계산하고 리포트에 경고.
- **플랫 밴드 부재:** 분산 분석으로 플랫 밴드가 안 나오면 거짓 양성을 만들지 말고 "플랫 밴드 없음"을 명확히 보고하라.
- **무거운 기호 계산:** 큰 `Q` 에서 `sympy` 행렬식이 폭발할 수 있다. 타임아웃·수치 폴백을 항상 준비.
